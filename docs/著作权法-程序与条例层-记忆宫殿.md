# 著作权法 · 2F 程序与平台层 + B0 条例补丁层｜记忆宫殿房间表

> 依据：《著作权法》第 8、12、49、50、64 条；《信息网络传播权保护条例》（2013 修订）；《计算机软件保护条例》（2013 修订）；《民法典》网络侵权相关条文（1194–1197）  
> 定位：著作权馆二楼（平台通知删除与避风港）+ 地下室前半（软件/网络条例 B1–B8）  
> 司法解释后半见：`docs/著作权法-司法解释补丁层-记忆宫殿.md`（B9–B14）  
> 入口：3F·E1 条例电梯；上接 3F 侵权认定、4F 救济；权属回 1F  
> 数据：`docs/data/copyright-palace.json`

## 0. 本层总图

```text
3F·E1 电梯下来
    │
    ▼
【2F 程序与平台层】
  T1 登记台(12/软7) → T2 集体管理厅(8)
  → T3 通知书柜台(网14) → T4 删除·转送车间(网15)
  → T5 反通知回廊(网16/17)
  → T6 存储避风港(网22) → T7 搜索链接港(网23)
  → T8 错误通知责任(网24) → T9 民法典必要措施厅(1195/1197)
    │
    ▼
【B0 条例补丁层】
  B1 软件定义·思想不保护(软2/6)
  → B2 软件职务三情形(软13)（对照1F·O7）
  → B3 合法复制品所有人(软16) → B4 学习研究例外(软17)
  → B5 软件侵权与罚款(软24) → B6 善意持有人(软30)
  → B7 有限表达不侵权(软29) → B8 可避开技术措施(著50)
```

**本层只解决一件事：** 平台要不要担责、通知删除怎么走、软件条例补哪些著作权法没写细的口。  
用户直接侵权认定 → 回 3F；赔多少 → 上 4F；权属 → 回 1F。

口诀：**通知删除冲流水线，明知应知出避风港；软件另开地下室。**

## 1. 2F｜程序与平台层

### 房间 T1｜登记台 · 著作权法 12② + 软件条例 7

| 项 | 内容 |
|----|------|
| **空间** | 二楼服务台：作品登记章 / 软件登记章；告示「初步证明，非生效条件」 |
| **感官** | 盖章咚；证书纸涩 |
| **场景钩** | 要不要登记再起诉？软件登记有什么用？ |
| **30 秒口述** | 作者等可办理作品登记。软件可向认定机构登记，登记证明文件是登记事项的**初步证明**。登记便利举证，**不是**权利产生条件（回 1F·O1） |
| **案例锚** | **登记举证** · **初步证明** · **非生效** |
| **关联** | ↔ O1、O6；软件权属细拆→B2 |

### 房间 T2｜集体管理厅 · 第 8 条

| 项 | 内容 |
|----|------|
| **空间** | 授权柜台：集体管理组织以自己名义主张权利；墙上挂使用费收取/转付公示板 |
| **感官** | 授权书撕联；公示屏冷光 |
| **场景钩** | KTV、广播、平台批量使用音乐，谁来收许可费、谁来起诉 |
| **30 秒口述** | 权利人可授权集体管理组织行使权利；依法设立的集体管理组织是非营利法人，被授权后可以**自己名义**主张权利，并可作为当事人参加诉讼、仲裁、调解。使用费标准协商不成可申请裁决等（细则另规） |
| **案例锚** | **自己名义主张** · **非营利法人** · **批量许可** |
| **关联** | ↔ O4 许可；收费争议细则继续挂后室 |

### 房间 T3｜通知书柜台 · 信息网络条例 第 14 条

| 项 | 内容 |
|----|------|
| **空间** | 流水线起点：权利人递交书面通知，要求删除或断开链接 |
| **感官** | 打印纸热；材料清单三格（权属证明 / 定位信息 / 侵权陈述） |
| **场景钩** | 发现盗版视频在平台，先告用户还是先通知平台 |
| **30 秒口述** | 对提供**信息存储空间**或**搜索、链接**服务的网络服务提供者，权利人认为其服务所涉作品、表演、录音录像制品侵犯信息网络传播权，或权利管理电子信息被删改的，可提交**书面通知**，要求删除或断开链接。通知应包含条例要求的内容（权利人信息、定位信息、侵权情况陈述等） |
| **案例锚** | **书面通知** · **删除或断链** · **材料齐备** |
| **关联** | → T4；无效/瑕疵通知影响过错认定→T9 |
| **数字桩** | 桩 **1**＝一纸通知启动流水线 |

### 房间 T4｜删除·转送车间 · 第 15 条

| 项 | 内容 |
|----|------|
| **空间** | 传送带：接到通知 → **立即**删除/断链 → 同时转送服务对象；地址不明则网络公告 |
| **感官** | 急停铃「立即」；转送邮件嗖一声 |
| **场景钩** | 平台压了三天才删，算不算合格应对 |
| **30 秒口述** | 网络服务提供者接到通知书后，应当**立即**删除涉嫌侵权内容或断开链接，并**同时**将通知书转送服务对象；服务对象网络地址不明、无法转送的，应当将通知内容在信息网络上公告 |
| **案例锚** | **立即删除** · **同时转送** · **公告替代** |
| **关联** | → T5、T6 |

### 房间 T5｜反通知回廊 · 第 16、17 条

| 项 | 内容 |
|----|------|
| **空间** | U 形回廊：用户交书面说明（反通知）→ 平台恢复 → 转送权利人；尽头牌子「权利人不得再通知删同一内容」 |
| **感官** | 回廊回声；恢复键咔哒 |
| **场景钩** | 用户主张合理使用或已获授权，要求恢复 |
| **30 秒口述** | 服务对象认为不侵权的，可提交书面说明要求恢复。平台接到后应**立即恢复**，并将说明转送权利人；权利人**不得再通知**删除该同一内容（此后纠纷常转诉讼） |
| **案例锚** | **反通知** · **恢复** · **不得再通知** |
| **关联** | ↔ S1 合理使用；→ 3F/4F 实体战 |

### 房间 T6｜存储避风港 · 第 22 条

| 项 | 内容 |
|----|------|
| **空间** | 港湾闸门五道条件灯（标示信息、未改变作品、不知道也没有合理理由应知、未直接获益、接到通知后删除等——口述抓主干） |
| **感官** | 绿灯齐亮才进港；「应知」红旗在风里猎猎 |
| **场景钩** | UGC 平台被诉：我们只是存储空间 |
| **30 秒口述** | 为服务对象提供信息存储空间，供其通过信息网络向公众提供作品等，并具备条例规定条件的（含标示身份、未改变作品、**不知道也没有合理理由应当知道**侵权、未直接获得经济利益、接到通知后删除等），**不承担赔偿责任**。避风港免的是赔，不是否认用户可能侵权 |
| **案例锚** | **存储空间** · **不知道亦不应知** · **通知后删除** |
| **关联** | ↔ T7；红旗/应知→T9；侧签：自动接入/缓存等另有 20、21 条港口 |
| **商标对照** | ≈ 商标 B6 网络补丁的「平台过错」思路 |

### 房间 T7｜搜索链接港 · 第 23 条

| 项 | 内容 |
|----|------|
| **空间** | 搜索灯塔：断链可免赔；灯塔底座写「明知或应知 → 共同侵权」 |
| **感官** | 断链剪刀声；红旗灼手 |
| **场景钩** | 聚合盗版链接、热播影剧榜单导流 |
| **30 秒口述** | 提供搜索或链接服务的，接到通知后依条例断开侵权链接的，不承担赔偿责任；但是，**明知或者应知**所链接的作品等侵权的，应当承担**共同侵权责任** |
| **案例锚** | **断链免赔** · **明知应知** · **共同侵权** |
| **关联** | ↔ T6；→ T9 |
| **数字桩** | 桩 **2**＝明知/应知对撞门 |

### 房间 T8｜错误通知责任 · 第 24 条

| 项 | 内容 |
|----|------|
| **空间** | 误伤柜台：错误删除/断链给服务对象造成损失 → 权利人赔 |
| **感官** | 误删警报刺耳 |
| **场景钩** | 权利人滥发通知搞垮对手正当内容 |
| **30 秒口述** | 因权利人的通知导致网络服务提供者错误删除或错误断开链接，给服务对象造成损失的，**权利人应当承担赔偿责任** |
| **案例锚** | **错误通知** · **权利人赔偿** · **通知审慎** |
| **关联** | ↔ T3；发通知前核权属与定位 |

### 房间 T9｜民法典必要措施厅 · 第 1195、1197 条（口径）

| 项 | 内容 |
|----|------|
| **空间** | 电梯旁民法典铜牌：通知后未及时必要措施 → 就损害扩大部分与用户承担连带；**知道或应当知道**仍未必要措施 → 连带 |
| **感官** | 铜牌冰凉；「红旗」意象与条例应知合流 |
| **场景钩** | 没发通知但热门盗版置顶；或通知后拖着不删 |
| **30 秒口述** | 网络用户利用网络服务侵权：权利人有权通知平台采取删除、屏蔽、断开链接等必要措施；平台接到通知未及时采取必要措施的，就**损害扩大部分**与用户承担连带责任。平台**知道或者应当知道**用户侵权未采取必要措施的，与用户承担连带责任。条例避风港与民法典过错规则要一起看，不能死背「无通知即无责」 |
| **案例锚** | **必要措施** · **损害扩大部分** · **知道或应当知道** |
| **关联** | ↔ T6、T7；细则裁判口径继续积累 |
| **对照** | 条例偏「通知删除程序」；民法典钉「过错 + 必要措施」 |

**2F 闭眼口诀：**  
通知（14）→ 立即删并转送（15）→ 反通知恢复（16/17）→ 存储/链接避风港（22/23）→ 错通知自担（24）→ 明知应知与民法典必要措施（1195/1197）。

## 2. B0｜条例补丁层（软件为主）

### 房间 B1｜软件定义·思想不保护 · 软件条例 2、6

| 项 | 内容 |
|----|------|
| **空间** | 地下室入口：程序+文档货架；思想/处理过程/操作方法/数学概念贴红封条 |
| **感官** | 源码与目标码挂同一吊牌「同一作品」 |
| **场景钩** | 只抄了功能流程没抄代码？ |
| **30 秒口述** | 软件＝计算机程序及其有关文档；源程序与目标程序为同一作品。保护不延及开发软件所用的**思想、处理过程、操作方法或数学概念**等（与 3F·S4 思想表达二分同向） |
| **案例锚** | **程序+文档** · **源目标同一** · **思想不保护** |
| **关联** | ↔ O2、S4、R1 |

### 房间 B2｜软件职务三情形 · 软件条例 13（对照 1F·O7）

| 项 | 内容 |
|----|------|
| **空间** | 三扇门：本职明确目标 / 本职可预见结果 / 主要使用单位物质技术条件并由单位承担责任 → 著作权归单位 |
| **感官** | 门比著作权法 18 更「软件专用」 |
| **场景钩** | 员工软件权属争的是著 18 还是软 13 |
| **30 秒口述** | 自然人在单位任职期间开发的软件，有下列情形之一的，著作权由单位享有并可给奖励：①针对本职工作中明确指定的开发目标；②开发是本职活动可预见或自然的结果；③主要使用单位资金、专用设备、未公开专门信息等物质技术条件并由单位承担责任。办案时与著作权法 18 对照，软件案优先摸清条例三情形 |
| **案例锚** | **本职目标** · **可预见结果** · **物质技术条件** |
| **关联** | ↔ O7；登记→T1 |
| **易混点** | 著 18「特殊职务」列举软件；软 13 把「本职目标/可预见结果」写得更操作化 |

### 房间 B3｜合法复制品所有人 · 软件条例 16

| 项 | 内容 |
|----|------|
| **空间** | 用户工具箱：装入装置 / 备份 / 必要修改；备份不得给他人；修改件原则上不得提供第三方 |
| **感官** | 备份光盘冰凉；「不得提供第三方」封条 |
| **场景钩** | 正版用户改了适配层又分发给客户 |
| **30 秒口述** | 合法复制品所有人可以：①按需要装入计算机等装置；②为防损坏制作备份（不得提供他人，丧失所有权时销毁备份）；③为实际应用或改进性能做必要修改，但除合同另有约定外，未经许可不得向第三方提供修改后的软件 |
| **案例锚** | **装入** · **备份** · **必要修改不外传** |
| **关联** | ↔ B6；最终用户场景高频 |

### 房间 B4｜学习研究例外 · 软件条例 17

| 项 | 内容 |
|----|------|
| **空间** | 小书房灯：为学习研究设计思想和原理，可安装显示传输存储，不经许可不付酬 |
| **感官** | 台灯暖，书房外禁止商业灯箱 |
| **场景钩** | 逆向工程、课堂教学演示 |
| **30 秒口述** | 为学习和研究软件内含的设计思想和原理，通过安装、显示、传输或存储等方式使用软件的，可以不经许可、不支付报酬 |
| **案例锚** | **学习研究** · **思想原理** · **非商业边界慎划** |
| **关联** | ↔ S1；与破解技术措施边界→B8 |

### 房间 B5｜软件侵权与罚款 · 软件条例 24

| 项 | 内容 |
|----|------|
| **空间** | 处罚价目墙：复制/发行出租信息网络传播；破坏技术措施；删改权利管理信息；擅自转让许可 |
| **感官** | 「每件 100 元」小字；「货值 1～5 倍」大字（2013 修订口径） |
| **场景钩** | 批量盗版软件、提供破解补丁 |
| **30 秒口述** | 未经许可复制或部分复制；向公众发行、出租、通过信息网络传播；故意避开或破坏技术措施；故意删改权利管理电子信息；转让或许可他人行使著作权人的软件著作权等，承担民事责任；损害公益还可行政查处。罚款口径（2013）：有复制或发行出租网传等行为，可并处每件 100 元或货值金额 1 倍以上 5 倍以下；有技术措施、权利管理信息、擅自转让许可等行为，可并处 20 万元以下罚款（以现行条例文本为准） |
| **案例锚** | **软件侵权清单** · **行政罚款** · **技术措施** |
| **关联** | ↔ R3、R5、F2；赔偿仍回著 54 |

### 房间 B6｜善意持有人 · 软件条例 30

| 项 | 内容 |
|----|------|
| **空间** | 最终用户窗：不知道也没有合理理由应当知道是侵权复制品 → **不赔**，但应停止使用并销毁；重大损失可付合理费用后继续使用 |
| **感官** | 销毁粉碎声；「合理费用」收据 |
| **场景钩** | 公司采购了盗版办公软件，称不知情 |
| **30 秒口述** | 软件复制品持有人不知道也没有合理理由应当知道是侵权复制品的，不承担赔偿责任；但应停止使用、销毁。若停止使用并销毁将造成重大损失，可在向著作权人支付合理费用后继续使用 |
| **案例锚** | **善意不赔** · **停止销毁** · **合理费用续用** |
| **关联** | ↔ F5/S3；免赔不免停的软件版 |
| **数字桩** | 桩 **0**＝善意持有赔偿落空（仍可能停用销毁） |

### 房间 B7｜有限表达不侵权 · 软件条例 29

| 项 | 内容 |
|----|------|
| **空间** | 窄门：可供选用的表达方式有限而相似 → 不构成侵权 |
| **感官** | 门窄，两人侧身才能过 |
| **场景钩** | 功能相同、接口惯例导致代码结构接近 |
| **30 秒口述** | 软件开发者开发的软件，由于可供选用的表达方式有限而与已经存在的软件相似的，不构成对已有软件著作权的侵犯 |
| **案例锚** | **有限表达** · **必要相似** · **非侵权** |
| **关联** | ↔ S4、R1；思想表达与合并原则的软件口 |

### 房间 B8｜可避开技术措施 · 著作权法 第 50 条

| 项 | 内容 |
|----|------|
| **空间** | 锁旁合法钥匙架：教学科研、无障碍、执行公务等可避开情形（不得向他人提供避开技术、装置、部件） |
| **感官** | 合法钥匙冰凉；非法工具箱上锁 |
| **场景钩** | 学校为课堂教学解密、为阅读障碍者提供 |
| **30 秒口述** | 法律列举情形下可以避开技术措施，但不得向他人提供避开技术措施的技术、装置或部件，也不得侵犯权利人其他权利。具体情形以第 50 条列举为准；与 49 条禁止破坏技术措施对照记 |
| **案例锚** | **法定可避开** · **禁止提供工具** · **对照49** |
| **关联** | ↔ R5；条例与司法解释细节随案积累 |
